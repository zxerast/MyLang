# Семантика языка MyLang

## Стратегия вычисления

**Call-by-value**: аргументы функции вычисляются до вызова и передаются как копии.

```
void modify(int x) {
    x = 99;   // изменяет только локальную копию
}

int a = 5;
modify(a);
// a всё ещё равно 5
```

## Семантика присваивания

**Value semantics**: присваивание всегда копирует значение.

```
Point p1 = Point { x: 1, y: 2 };
Point p2 = p1;   // копия
p2.x = 99;
// p1.x всё ещё 1
```

## Инициализация переменных

Использование неинициализированной переменной — автоматически инициализирует значение по умолчанию своего 
типа.

Значения по умолчанию (default):

int -> 0
float -> 0.0
string -> "NULL"
bool -> false
char -> '0'
type[3] -> [default, default, default]

```
int x;        // OK 
int y = 3;    // OK
int z = w + y     // Ошибка компиляции w - необъявлена
int w = x + y // OK, эквивалентно: 0 + 3
```

### Значения по умолчанию составных типов

Для структур и пользовательских типов значения по умолчанию
определяются значениями их полей.

struct Point {
    int x;
    int y;
}

// default(Point) = { x: 0, y: 0 }

### Изменение значений по умолчанию

Язык допускает изменение значений по умолчанию для полей
пользовательских типов через обращение к типу:

Point.x = 10;

После этого все последующие неявные инициализации используют
обновлённое значение:

Point p;   // p.x == 10

Изменение значений по умолчанию:
- влияет только на последующие инициализации
- не изменяет уже существующие объекты

## Время жизни переменных

Переменная существует в пределах блока `{ }`, в котором объявлена.
При выходе из блока переменная уничтожается.

```
int x = 1;
{
    int y = 2;
    // x и y доступны
}
// y недоступен, x доступен
```

## Область видимости

Язык использует **лексическую область видимости** (lexical scoping).

Правила разрешения имён:
- Идентификатор разрешается в ближайшей охватывающей области видимости
- **Shadowing допускается**: внутреннее объявление скрывает внешнее
- Использование идентификатора до его объявления — ошибка компиляции
- Одно и то же имя можно дважды объявить в одной области видимости, будет происходить shadowing по очереди 
объявлений

```
int x = 1;
{
    int x = 2;   // OK: shadowing
    // здесь x == 2
}
// здесь x == 1
```

```
int x = y + 1;   // ОШИБКА: y не объявлен
int y = 5;
```

```
int x = 1;
int x = 2;       // новый x перекрывает старое значение
```

## Порядок объявлений на верхнем уровне

На верхнем уровне (вне функций) все объявления видны сразу же, двухпроходный анализ.

```
int main(){
    int x = 9, y = 2;
    Add(x, y);
    return 0;
}

void Add(int a, int b){     //  Add объявлена позже, но её вызов в main допускается
    print(a + b);
}
```

## Управление потоком

### if / else

Условие обязано иметь тип `bool`. Неявного приведения к `bool` нет.

```
if (x > 0) { ... }       // OK
if (x) { ... }           // ОШИБКА, если x типа int
```

### while

Условие обязано иметь тип `bool`.
`break` и `continue` допустимы только внутри тела цикла.

### return

В функции с типом возврата отличным от `void` — `return` обязан присутствовать на всех путях выполнения 
(проверяется компилятором).

В `void`-функции `return` без значения опционален.

```
int sign(int x) {
    if (x > 0) { return 1; }
    if (x < 0) { return -1; }
    return 0;   // обязателен
}
```

## Порядок вычисления операндов

Операнды бинарных операторов вычисляются **слева направо**.

## Побочные эффекты

Функции могут иметь побочные эффекты: ввод/вывод, изменение переменных через параметры — нет (передача по 
значению), но глобальные переменные изменять можно.

## Ошибки времени выполнения

При возникновении ошибки программа завершается с выводом в `stderr`:

```
runtime error: <сообщение> at line <N>
```

Обязательно обрабатываются:

| Ситуация | Сообщение |
|----------|-----------|
| Деление на ноль | `division by zero` |
| Выход за границы массива | `index out of bounds: index N, size M` |
| pop() из пустого массива | `pop from empty array` |

## Встроенные функции

| Функция | Сигнатура | Описание |
|---------|-----------|----------|
| `print` | `void print(T, ...)` | вывод значения + перенос строки |
| `input` | `auto input()` | чтение ввода из stdin |
| `len` | `int len(string)` или `int len(T[i])` или `int len(T[])` | длина строки или массива |
| `exit` | `void exit(int)` | завершение программы с кодом |
| `panic` | `void panic(string)` | аварийное завершение с сообщением |
| `push` | `void push(T, T[])` | добавить элемент в конец динамического массива |
| `pop` | `T pop(T[])` | удалить и вернуть последний элемент (runtime error если пуст) |

`print` - вариадичная функция, принимает сколько угодно параметров и печатает их через ',', принимает 
любой тип — перегрузка встроенная, не пользовательская.

## Пространства имён

Объявления внутри `namespace` доступны снаружи через `::`:

```
namespace Math {
    const int PI_INT = 3;
    int square(int x) { return x * x; }
}

int r = Math::square(5);
```

Внутри `namespace` можно обращаться к своим членам без префикса.
Вложенные пространства имён поддерживаются.

## Неявные приведения типов

В контекстах присваивания, передачи аргументов и бинарных операций допускаются безопасные расширяющие 
(widening) приведения:

- `int8` → `int16` → `int32` → `int64`
- `uint8` → `uint16` → `uint32` → `uint64`
- `float32` → `float64`
- Любой целый тип → `float64`
- `float` -> `int` - запрещён, из-за урезания точности 

```
int32 x = 5;
int64 y = x;        // OK: widening
float z = x + y;    // OK: int64 → float64
```

В бинарных операциях с разными числовыми типами оба операнда приводятся к большему типу:
- `int32 + int64` → `int64`
- `int + float` → `float`

Сужающие приведения (narrowing) требуют явного `cast<T>()`.

## Контекстная типизация

Контекстная типизация — это механизм, при котором тип выражения
может быть выведен или уточнён из ожидаемого типа контекста.

Контексты, в которых применяется:
- правая часть присваивания
- аргументы функции
- возвращаемое значение функции
- элементы массивов

В этих контекстах допускаются неявные расширяющие приведения типов.

Примеры:

int x = 5;          // литерал 5 имеет тип int32 по умолчанию, ничего не меняется
float y = 5;        // 5 неявно приводится к float64
auto z = 5;         // контекст отсутствует, тип выводится из значения напрямую

void f(float a) { ... }
f(10);              // 10 → float64

int arr[3] = [1, 2, 3];   // элементы приводятся к типу массива

## Модули

Каждый исходный файл (`.lang`) является модулем.

### import

```
import "math.lang";
```

- Все `export`-объявления из указанного файла становятся доступны в текущем модуле
- Импорт помещается в начало файла, до любых объявлений
- Циклические импорты — ошибка компиляции
- Путь разрешается относительно текущего файла

### export

```
export int add(int a, int b) { return a + b; }
export struct Point { int x; int y; }
export const int MAX = 100;
```

- Без `export` объявление видно только внутри своего файла
- `export` допускается только на верхнем уровне
- `main` не требует `export`

## Взаимодействие с C

При вызове функций из C применяются следующие правила маршалинга: (неподдерживаемые типы привести к 
поддерживаемым)

Строки (string): MyLang автоматически передает в C-функцию указатель на const char*. Строка гарантированно 
дополняется нуль-терминатором \0 во временном буфере перед вызовом.

Массивы: При передаче массива (T[] или T[N]) в C-функцию, передается прямой указатель на его первый 
элемент в памяти. Поскольку в MyLang используется семантика значений, передается указатель на локальную 
копию массива.

Тип ptr (любой указатель из C): Передается как есть в типе uint64 как числовое значение адреса.

Управление памятью: MyLang не отслеживает время жизни объектов, созданных внутри C-библиотек. Если 
C-функция возвращает ptr, программист обязан вручную вызвать соответствующую функцию очистки из C 
(например, fclose или free)

## Точка входа

Программа обязана содержать функцию `main` с типом возврата `int`:

```
int main() {
    return 0;
}
```

Возвращаемое значение передаётся как код завершения процесса.
