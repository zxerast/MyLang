# Система типов MyLang

## Встроенные типы

| Имя | Размер | Описание |
|-----|--------|----------|
| `int` | 64 бит | знаковое целое (псевдоним `int64`) |
| `uint` | 64 бит | беззнаковое целое (псевдоним `uint64`) |
| `int8` | 8 бит | знаковое целое |
| `int16` | 16 бит | знаковое целое |
| `int32` | 32 бит | знаковое целое |
| `int64` | 64 бит | знаковое целое |
| `uint8` | 8 бит | беззнаковое целое |
| `uint16` | 16 бит | беззнаковое целое |
| `uint32` | 32 бит | беззнаковое целое |
| `uint64` | 64 бит | беззнаковое целое |
| `float` | 64 бит | вещественное (псевдоним `float64`) |
| `float32` | 32 бит | вещественное (IEEE 754) |
| `float64` | 64 бит | вещественное (IEEE 754) |
| `bool` | 8 бит | булево значение (`true` / `false`) |
| `char` | 8 бит | ASCII-символ |
| `string` | — | строка (длина + байты) |
| `void` | — | отсутствие значения, только как тип возврата функции |

## Составные типы

### Фиксированный массив

```
T[N]
```

- `T` — тип элемента, `N` — числовое значение, литерал, переменная, возвращаемое значение (размер фиксирован)
- Размер является частью типа: `int[3]` и `int[4]` — разные типы, привидение запрещено
- Индексирование: `arr[i]`, индекс типа `int` или `uint`
- Вложенные: `int[3][4]` — массив из 3 элементов, каждый — массив из 4 int

### Динамический массив

```
T[]
```

- `T` — тип элемента, размер определяется во время выполнения
- `int[]` и `int[3]` — разные типы; литерал массива неявно приводится к динамическому
- Индексирование: `arr[i]`, индекс типа `int` или `uint`
- Создание: литерал массива или встроенные функции
- Вложенные: `int[][]` — динамический массив динамических массивов

```
int[] arr = [1, 2, 3];       // создание из литерала
string[] names = [];          // пустой динамический массив
push(4, arr);                 // arr = [1, 2, 3, 4]
int last = pop(arr);          // last = 4, arr = [1, 2, 3]
int[][] matrix = [[1, 2], [3, 4]];  // вложенный массив
```

- Динамические массивы хранятся в куче (heap)
- При присваивании и передаче в функцию — копируются (value semantics)

### Структура

```
struct Point {
    int x;
    int y = 9;
}
```

- Поля хранятся последовательно в памяти
- Доступ к полю: `p.x` - значение экземпляра, `Point.y` - значение по умолчанию
- Структуры передаются по значению (копирование)

## Номинальная типизация

Два типа совместимы только если их прообразы совпадают.

```
type Meters = int;
type Seconds = int;

Meters m = 10;
Seconds s = m;   // OK: Meters == Seconds == int
```

Синоним типа — это альтернативное имя, а не новый тип:

```
type Meters = int;
int x = 10;
Meters m = x;   // OK: Meters это тот же int
```

## Неявные приведения (widening)

Безопасные расширяющие преобразования выполняются автоматически:

```
int8 → int16 → int32 → int64 (int)
uint8 → uint16 → uint32 → uint64 (uint)
float32 → float64 (float)
любой int → float64
любой uint → float64
```

```
int32 x = 5;
int64 y = x;       // OK: неявное расширение int32 → int64
float z = x;        // OK: неявное расширение int32 → float64
```

Сужающие преобразования (narrowing) и все прочие приведения по-прежнему требуют явного `cast<T>(expr)`:

```
int64 big = 1000;
int8 small = big;            // ОШИБКА: narrowing
int8 small = cast<int8>(big); // OK: явное усечение
```

Допустимые приведения:

| От | К | Комментарий |
|----|---|-------------|
| любой int | любой int | narrowing/widening, усечение при narrowing |
| любой int | float32/float64 | расширение |
| float32/float64 | любой int | усечение дробной части |
| float32 | float64 | расширение |
| float64 | float32 | усечение |
| любой int | bool | 0 → false, остальное → true |
| bool | int | false → 0, true → 1 |
| char | int | значение байта |
| int | char | младший байт |
| bool | string | false -> "false", true -> "true" |
| string | bool | любая строка -> true, "NULL" -> false |

Приведения между `string` и числовыми типами не допускаются.

## Мутабельность

По умолчанию переменная мутабельна. `const` запрещает переприсваивание:

```
int x = 5;
x = 10;          // OK

const int MAX = 100;
MAX = 200;        // ОШИБКА: переменная константна
```

`const` на параметры функции распространяется

## Вывод типов (auto)

Ключевое слово `auto` позволяет не указывать тип переменной явно — он выводится из выражения инициализации:

```
auto x = 5;             // int
auto y = 3.14;          // float
auto s = "hello";       // string
auto flag = x > 0;      // bool
auto arr = [1, 2, 3];   // int[] (динамический массив)
```

Правила:
- `auto` обязывает наличие инициализатора (`auto x;` — ошибка)
- Выведенный тип фиксируется при объявлении, привидение далее разрешено
- `const auto` допускается
- `auto` допускается в параметрах функций и полях структур при наличии значений по умолчанию

## Тип числовых литералов

Числовые литералы имеют **контекстный тип**: если из контекста известен ожидаемый числовой тип, литерал принимает его. Без контекста применяются типы по умолчанию:

- Целый литерал (`123`) — по умолчанию `int` (`int32`)
- Вещественный литерал (`3.14`) — по умолчанию `float` (`float64`)

Контекст применяется при:
- объявлении переменной с явным типом: `int8 x = 42`
- передаче аргумента в функцию: `foo(42)` где параметр `int8`
- возврате из функции: `return 42` в функции с `int8`
- присваивании: `x = 42` где `x` имеет тип `int8`

Целый литерал принимает любой целый или вещественный тип из контекста. Вещественный литерал — только вещественный.

```
int x = 5;          // OK: 5 → int32 (по умолчанию)
int8 a = 42;        // OK: 42 → int8 (из контекста)
int64 b = 9000;     // OK: 9000 → int64 (из контекста)
float32 f = 3.14;   // OK: 3.14 → float32 (из контекста)
int8 c = 3.14;      // ОШИБКА: вещественный литерал не принимает целый тип
```

## Строки

- Тип `string` — неизменяемая последовательность байт (UTF-8)
- Операции:
  - Конкатенация: `s1 + s2`
  - Длина: встроенная функция `len(s)` возвращает `int`
  - Сравнение: `==`, `!=`
- Индексирование строк поддерживается

## Переполнение

Переполнение целых чисел — поведение как в C: для знаковых не определено (wraparound на x86-64), для беззнаковых — wraparound по модулю 2^N.

## Void

`void` допускается только как тип возврата функции. Переменных типа `void` не существует.
